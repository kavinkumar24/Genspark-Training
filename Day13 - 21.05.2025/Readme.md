## Model

## Interface

- Defines a contract without implementation

## Repository Pattern

- Separates data access logic from business logic by providing a consistent way to interact with data sources

## Abstract class

- A partially implemented class that can't be instantiated on its own

## Concrete Class

- A fully implemented class that can be instantiated and provides actual behavior defined by abstract class
